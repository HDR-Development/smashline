1//! `smashline` is an open source project aimed at making code modifications to Super Smash Bros.
2//! Ultimate an easier experience.
3//!
4//! You can find some top-level documentation on how SSBU manages character code and scripts on this
5//! page, and you can also find more nuanced information about how this project functions on the
6//! various other crate documentation pages.
7//!
8//! # Scripting Mechanisms
9//! Based on some experience in modding games released on the switch, it appears that Nintendo most
10//! likely offers assistance in some form, whether it be middleware or direct assistance, in
11//! getting games and tools running with [Lua](https://www.lua.org/) a scripting language that
12//! is pretty easy to integrate with and decently fast as well.
13//!
14//! SSBU makes use of Lua scripts heavily for its battle object scripting code, with some slight
15//! caveats:
16//! - `fighter` and `weapon` scripts are converted from compiled Lua byte-code into C++ source code
17//! via an in-house transpiler called `lua2cpp`.
18//! - `item` animation command (referred to as either AnimCMD or ACMD) scripts are kept as compiled
19//! Lua byte-code in the game's romfs data and their status scripts are transpiled the same as
20//! `fighter` and `weapon` scripts.
21//!
22//! Using Lua scripts in this fashion likely was a choice for quick development and easy integration
23//! with other tools. Since Lua is a drop-in utility, the Lua scripts themselves were likely
24//! auto-generated via a visual hitbox, effect, and sound effect placement/timeline tool.
25//! Unfortunately, modding tools are not that advanced yet so we must resolved to writing our
26//! scripts directly in code.
27//!
28//! SSBU also uses Lua scripts for *some* of its menus, however that use case is not supported by
29//! this project.
30//!
31//! ## Animation Commands (AnimCMD or ACMD) and Lua coroutines
32//! Another great reason to use Lua for this kind of process is that Lua comes with built-in
33//! coroutine support, which makes a lot of sense to use in the case of a move that will be playing
34//! out over the course of multiple frames.
35//!
36//! Here is an example Lua ACMD script, with documented comments about where the coroutines
37//! will yield and how exactly they unyield:
38//! ```lua
39//! -- Start of the script
40//!
41//! -- Yield this coroutine, setting the unyield condition to once the frame timer is >= 5
42//! frame(5)
43//!
44//! -- is_excute will be documented later
45//! if is_excute() then
46//!     WorkModule.on_flag(EXAMPLE_FLAG)
47//! end
48//!
49//! -- Yield this coroutine, setting the unyield condition to once the frame
50//! -- timer is >= (current_frame + 10)
51//! wait(10)
52//! if is_excute() then
53//!     AttackModule.clear_all()
54//! end
55//!
56//! -- Yield this coroutine, setting the unyield condition to once the frame timer is >= 30
57//! frame(30)
58//! if is_excute() then
59//!     WorkModule.off_flag(EXAMPLE_FLAG)
60//! end
61//! ```
62//!
63//! This makes performing operations on specific frames a lot easier to both read and write,
64//! since they are expressed as a set of procedural instructions instead of logic that might
65//! be operating once on every frame.
66//!
67//! ### AnimCMD Categories
68//! To further split up the logic into more maintainable components, there are four different
69//! categories of ACMD scripts:
70//! - `game` scripts
71//!   
72//!   `game` scripts operate on the core state of the object. Here you will find hitbox placements,
73//! state manipulation, occasional button checks, etc.
74//!
75//!   An integral part of the animation is also exclusively controlled in these scripts: frame
76//! pacing. Via calls to `FT_MOTION_RATE`, these scripts can change the speed of the animation and
77//! how fast the script playback is. For this reason, calls to effect and sound libraries are
78//! off-handed to the other scripts as a nice separation. The speed of those scripts are also
79//! impacted by these calls.
80//! - `effect` scripts
81//!   
82//!   `effect` scripts are purely visual, although changing them may lead to changes in how the
83//! RNG generators work since some effects generate RNG values. Sword trails, singular effects,
84//! flashes, screen backgrounds, etc. are usually all controlled by effects. They are not always
85//! in the `effect` scripts, but for animations' associated effects that is where you will find
86//! them.
87//! - `sound` scripts
88//!    
89//!   `sound` scripts make calls to the game's sound module and libraries to either play sound
90//! effects or pick a sound effect to play from a sound effect bank.
91//! - `expression` scripts
92//!
93//!   `expression` scripts are basically the catch-all for everything else. Changing how an object
94//! might display an interaction with sloped ground or how rumble is applied will happen in these
95//! scripts.
96//!

117#[skyline::main(name = "smashline")]
118pub fn main() {
119    create_agent::install_create_agent_hooks();
120    create_agent::install_create_agent_share_hooks();
121    create_agent::install_status_create_agent_hooks();
122    nro_hook::install();
123    interpreter::install();
124    runtime_reload::install();
125    state_callback::install_state_callback_hooks();
126    callbacks::install_callback_hooks();
127    unwind::install_unwind_patches();
128    cloning::install();
129    params::install_param_hooks();
130    effects::install_effect_transplant_hooks();
131
132    std::panic::set_hook(Box::new(|info| {
133        let location = info.location().unwrap();
134
135        let msg = match info.payload().downcast_ref::<&'static str>() {
136            Some(s) => *s,
137            None => match info.payload().downcast_ref::<String>() {
138                Some(s) => &s[..],
139                None => "Box<Any>",
140            },
141        };
142
143        let err_msg = format!("smashline has panicked: '{}', {}", msg, location);
144        skyline::error::show_error(
145            69,
146            "Smashline has panicked! Please open Details and post an issue at https://github.com/HDR-Development/smashline.",
147            err_msg.as_str(),
148        );
149    }));
150}
